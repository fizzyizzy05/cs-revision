The CPU

  • The CPU is the component of a computer that enables it to carry out instructions.
  • It also contains it's own components - these include the Control Unit, Arithmetic Logic Unit, Dedicated Registers and various types of buses including Address, Data and Control.
  • The CPU also contains cache, a super-fast form of memory.

Memory Types

  • Registers: small pieces of memory used storing data located in the processor.
  • Random Access Memory (RAM): A form of memory that is used to store currently in-use programs and data
  • Storage: this could include Hard Disk Drives (HDDs) and Solid State Drives (SSDs). These are commonly used to store the computer's operating system as well as it's permanent files.

Buses

  • Address buses: used to carry the address of where data is going to or from.
  • Data buses: used to carry data from one place to another.
  • Control buses: used to carry control signals from the control unit to other parts of the system.
  • Important: buses don't actually store any information themselves: they just facilitate it's transfer between different parts of the system

Types of registers:

  • Control unit: used to decode instructions and control the flow of data through the buses.
  • Arithmetic Logic Unit (ALU): used to carry out arithmetic instructions and calculations.
  • Accumulator: used as temporary storage for the data being processed.
  • Program Counter (PC): used to store the address of the next instruction to be fetched.
  • Current Instruction Register (CIR): used to hold the current instruction being executed.
  • Memory Address Register (MAR): used to store the memory address where data in the MDR is about to be written to or read from.
  • Memory Data Register (MDR): used to hold the data that has just been read from or is about to be written to main memory.

Fetch, execute, decode (FED) cycle

  • When a program runs on a computer, it needs to be loaded into the main memory to be accessible by the processor through the FED cycle.
  • The fetch stage causes the next instruction to be fetched from memory.
  • The decode stage produces signals which control other parts of the computer such as the ALU.
  • The execute stage causes the instruction to actually be executed.

Opcodes and Operands

  • Word: a fixed size group of digits which is stored in the processor as a unit.
  • Opcodes are represented by the mnemonics like ADD, LOAD and HALT.
  • Operands are either an address or an actual value; this depends on the processing mode.

What affects computer performance

According to OCR, there are three main factors that can affect computer performance:

  • Clock speed: this determines how fast the CPU is ran and how quickly instructions are ran.
  • Core count: this determines how many instructions can be ran simultaneously.
  • The amount and type of cache memory: this gives more space for data and instructions so RAM needs to be accessed less frequently.

Pipelining

  • Pipelining is a tactic used by some modern processors in order to improve performance.
  • It involves splitting up tasks into smaller parts and overlapping the processing of each part of the task.
  • It's commonly used in processors used in PCs in order to split up the FDE cycle like an assembly line.

Different computer architectures

Von neumann architecture

  • This architecture has the basic components of a computer processor specified with a shared bus used for both data and instructions.
  • This means it suffers from the von Neumann bottleneck because of the shared bus: this means either data or instructions can be retrieved, not both.
  • Only has a single Control Unit and Arithmetic Logic Unit.

Harvard architecture

  • This architecture has seperate memories for both instructions and data.
  • It features separate buses for instructions and data, meaning the address bus can be wider than the data bus.
  • It's used extensively in embedded systems.
  • Harvard architectures can be faster than Von Neumann due to the ability for data to be fetched in parallel.

Contemporary architecture

  • Contemporary processors often incorporate aspects of both von Neumann and Harvard architectures.
  • As well as the good features of Harvard and von Neumann architectures, contemporary processors could have some more advanced features such as pipelining, hyper-threading and integrated graphics with extra circuitry for graphics processing.
  • Most mobile processors and even the latest Intel chips also use a big.LITTLE architecture with some slower "efficiency cores" used for less intensive tasks and faster "performance cores" used for more intensive tasks. This aims to increase the efficiency of processors.
