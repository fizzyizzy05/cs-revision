Why might projects fail?

  • Lack of research.
  • Poor project leadership.
  • Not enough time to complete the project.
  • Poor testing and quality control.

How would we gather the requirements of the user/client?

  • Questionnaires: these are a set of questions that are given out to potential users for them to complete.
  • Interviews: these are face-to-face discussions between the developers and potential users. These will often entail asking questions to the users about the projects.
  • Observations: this is where the developers will observe users of the existing system to understand how it works.
  • Meetings: this is where a group of users sit down with an analyst to discuss the current system.
  • Document analysis: this is where existing documentation in the current system is explained.

Different stages of development

Analysis

  • What is the problem? What does the software need to accomplish to overcome it?
  • A problem needs to be defined before it can be solved and the requirements established.
  • There will need to be the consideration of many different factors. In the example of a data processing system, this could include the data, procedures, future and the problems with the current system.

Design

  • What will the system look like? what will it be made of?
  • Depending on the type of project, the systems designer may consider any of the following:
    • Processing
    • Data structures
    • Output/Input
    • User interface
    • Security
    • Hardware

Programming and testing

  • This is the part where the software is developed and checked to see if it works.
  • A system needs to be thoroughly tested before installation to ensure that all errors are discovered and corrected.
  • Programmers will often come up with a test plan, and testing their own code is natural - it usually runs in parallel with the creation of code.

Installation

  • This is the stage at which the software will be installed where it needs to be.
  • Coding and testing will be carried out, and errors will be traced and fixed.
  • When everything is considered satisfactory, the software will be installed on the user's machine.

Evaluation

  • This is where the development of the new software is evaluated to see if it's effective at it's job.
  • This will come in the form of a post-implementation review, which will focus on the following:
    • A comparison between the old system and the new one.
    • An assessment of each aspect of the system against pre-set criteria.
    • What errors were made durng the development phase.
    • Unexpected advantages and issues.
