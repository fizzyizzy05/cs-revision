Compression

  • Compression is used to save the user money by using less bandwidth/data. This is because compressed files download quicker.

Lossy compression

  • Lossy compression algorithms make a file smaller at the expense of the accuracy of the data it represents.
  • Lossy compression is used on images as:
    • Small detail loss is acceptable.
    • It's unlikely to be noticable.
    • The file size will work out smaller than alternatives.
  • Lossy compression permanently takes away from some of the information from the original file, it is irreversible.
  • Lossy compression is not used on text files as the removal of small ammounts of data will make the file unreadable.

Lossy compression may be used in the following situations:

  • Sound compression: MP3 can acheive powerful compression by stripping out the sounds which ears can't hear, as well as to use complex mathematical algorithms to futher reduce file sizes. This results in music files that are ~11x smaller than uncompressed music tracks.
  • Image compression: JPG files are the most common file format for sharing and storing digital images, especially on the web. It's ideal for images that use >256 colours, such as digital photos. When images are saved in JPG format, some data is lossed because the file is compressed. JPG files can be stored at various different quality settings, measured as a percentage of the original quality.

Lossless compression

  • Lossless compression preserves all of the information from the original file.
  • It's commonly used on files where losing data is not a viable option, such as code and text files.

There are two main methods of lossless compression:

  • Run-Length Encoding (RLE): this relies on consecutive sets of characters being the same. Each consecutive symbol and it's number of occurences could be represented by the symbol and its number of consecutive occurence. For example, AAABBBBBBBCC would be represented as A3B7C2. The compression ratio can be found by dividing the size of the compressed file by the size of the original.
  • Dictionary Encoding: this is where commonly occuring pieces of data/groups of characters are replaced by tokens. A dictionary is used to determine which tokens match which groups of characters. When the file is decompressed, the dictionarry is used to replace the tokens with the original text. As a consequence, the dictionary needs to be transmitted with the file.
  • The two may also be combined in order to increase the compression ratio.

Encryption

  • Encryption is the process of converting information or data into a code, especially to prevent unauthorised access.
  • The strongest encryption is unbreakable, even if it's attempted by a government agency.

Symmetric encryption

  • This is where the same key is used to decrypt and encrypt data.
  • Symmetric encryption would require key exchange, meaning both parties would need a copy of the key.
  • The key must not be transmitted over the internet as an eavesdropper might pick up on it.

Asymmetric Encryption

  • This where two different keys are used: one public key to encrypt the data and another private key to decrypt it.
  • Asymmetric is good for internet traffic as it removes the requirement of the key being sent, as there are two different ones.
