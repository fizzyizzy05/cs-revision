CISC and RISC processors

Complex Instruction Set Computing (CISC)

  • CISC processors have large instruction sets in order to accomplish as many tasks as possible with as few lines of code as possible.
  • The average program only uses around 20% of the instructions on offer by CISC processors.
  • Compared to RISC, less RAM is required to store instructions on CISC due to the code it produces being shorter.
  • CISC processors have the disadvantage of being much more expensive to manufacture due to the circuitry being complex.

CISC instructions

  • Instructions have a variable number of bytes.
  • Each instruction might take multiple cycles to complete. This is because the machine's hardware has complex instructions built in.
  • CISC processors often use more power than alternatives, meaning that they produce more heat. This makes CISC unsuitable for devices with limited power available.

Reduced Instruction Set Computing (RISC)

  • RISC processors tend to have less instructions available than CISC.
  • Each instruction performs a simple task, and take one clock cycle to execute. To do complex tasks, multiple instructions have to be combined
  • The combining of instructions required to complete complex tasks means more RAM is required to store machine code.

RISC instructions

  • Since each instruction has a fixed number of bytes, they take the same amount of time to complete.
  • Timings can be calculated more properly, meaning that pipelining is possible.
  • RISC has mostly replaced CISC in processor design. It's simpler processor design means it needs less transitors - producing less heat. Mobile phones, tablets, and some newer laptops also use it due to it's limited power use.

CISC & RISC compared

  • CISC processors take multiple cycles to complete instructions, RISC will only take one.
  • CISC instructions have a variable number of bytes, RISC will have a fixed number.
  • CISC has many more instructions available than RISC.
  • CISC integrated circuits are more expensive to produce.
  • CISC requires less RAM to complete instructions.
  • CISC produces more heat and uses more power compared to RISC.

Co-processors

  • A co-processor is an extra processor used to supplement the functions of the primary processor. An example use-case for one is rendering graphics.
  • Co-processors typically aren't general-purpose processors. This means they won't have the ability to fetch instructions, do I/O instructions, etc. Co-processors only typically carry out a limited range of functions.

Graphics Processing Units (GPUs)

  • GPUs are designed specifically with the purpose of processing and rendering graphics.
  • GPUs have a parallel architecture with thousands of small and efficient cores used to render graphics and handle multiple tasks simulatenously.
  • Personal computers tend to have GPUs on a graphics card, or alternatively embedded on the motherboard.
  • Adding a graphics card typically speeds up the rendering of 3D graphics as the GPU has specialist instructions/circuitry and have the ability to apply the same instructions to multiple
  • GPUs are increasingly a cost efficient way of increasing performance. GPUs have a significantly higher core count than most PCs, meaning that they are able to run highly parallelisable problems.

Multicore and Parallel systems

  • Multicore processor: a processor with more than one processing unit. Each core has the ability to independently process instructions simultaneously.
  • Multiple cores can be effective at speeding smaller problems, but often won't be enough for larger problems.

There are two approaches to multi-core processing of data:

  • Single Instruction, Multiple Data (SIMD): this is where the same instruction is applied to multiple pieces of data at once.
  • Multiple Instruction, Multiple Data (SIMD): this is where multiple instructions are applied to multiple pieces of data at once.
