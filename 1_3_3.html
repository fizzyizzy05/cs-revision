Protocols

Protocols are a set of rules for how devices communicate between each other. They may be implemented via software, hardware or both. Examples of protocols include:

  • HTTP: used for sending data over the internet.
  • DCHP: used for assigning IP addresses to devices
  • UDP: one of the core communication protocols of IP and used to send messages to hosts.

Protocols are neccessary to allow networks to communicate by ensuring all devices communicate in the same way.

TCP/IP Stack

The TCP/IP Stack is responsible for sending packets of data between hosts. It passes them down the stack, and adds neccesary data during the process.

The stack is made up of four layers, memorisable via the acronym ATIL:

  • Application: chooses the right protocol to send the data with based on the software or the data type.
  • Transport: uses TCP to split the data into packets and adds the packet number, total number of packets, and the port.
  • Internet: Alternatively known as the Network layer. Adds the IP addresses of the sender and the recipient, and packages data for transmission. Routers operate on this layer.
  • Link: adds in the MAC address and looks for the NIC to make sure the packets are in the right place.

Domain Name System

DNS is used to route a domain to an IP address. For example, if you typed in google.com into your browser's address bar, the browser will send the request for google.co.uk to a DNS server. The server will attempt to map google.co.uk to the right IP address, and if it finds a match, it will return it to the web browser. If it doesn't have the IP address, it will instead forward on the request to another DNS server which may have the IP address for this site. When a DNS server is found that has the IP address, the match of the site and IP are passed down to the other DNS servers.

Networks

Networks are two or more computers connected to a network. They are used for communication as well as the sharing of files, data and resources. They are also often used for backups, distributed computing and centralised administration.

Networking equipment:

  • Router: a hardware device that connects a network to the internet.
  • Modem: used to convert signals from an ISP such as Sky or BT and conversts them into signals local devices can use:
  • Switch: sends data to devices on a network. Uses a broadcast-style transmission where packets are sent to all devices.
  • Hub: similar functionality to a switch, but sends packets to the specific devices they are meant for.
  • Wireless Access Point: used to broadcast a network connection wirelessly.
  • Network Interface Card: used to connect devices to the network. Wireless Network Interface Cards can do this wirelessly when connected to a Wireless Access Point.

Circuit Switching

Circuit switching is the setup of a pathway for data to be sent. It is a direct link bewtwen devices for communication of data. It is generally considered to be outdated and wastes bandwidth when data is not sent, but does not require the reassembly of packets as they are already in the right order.

Packet Switching

Packet switching is the process of breaking up packets in order to send them across a network more efficiently. At each node on the network, the destination address is read, and the best route is found. Packet switching does not waste network bandwidth as the line does not need to stay open for the entire communication time. It is also more secure as data is split across multiple different packets, meaning that many pieces of the data would be needed in order to obtain a readable message.

Firewalls

Firewalls are hardware or software devices used to secure networks. They scan incoming and outgoing data packets and block anything that is suspicious or unusual. They are used to prevent unauthorised access to data or data theft.

Hackers

A hacker is someone who uses computer code to gain access to a computer system. They are generally categorised in colours of hat:

  • Black hat: anyone who hacks into a system for malicious purposes including unauthorised access, the spread of malware, or data theft.
  • White hat: anyone who hacks into a system in order to test its permission with prior permission.
